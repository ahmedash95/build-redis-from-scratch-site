### كتابة Main.go

أثناء كتابة المشروع، لن نستخدم أي مكتبات خارجية، وذلك لأن كل ما نحتاجه متوفر في Go نفسها. كل ما تحتاجه هو أن تبدأ مشروع Go جديدًا وتجهز الـ Main.go.

```go
package main

import (
    "fmt"
)

func main() {
    fmt.Println("Hello World")
}
```

### تجهيز الـ Client

سنشير دائمًا إلى قاعدة البيانات الداخلية التي سنكتبها بـ "الخادم"، وذلك لأنها قاعدة بيانات خادم مثل خادم MySQL أو Redis.

وبما أن الخادم دائمًا يحتاج إلى "عميل" للتواصل معه، فإننا سنحتاج أيضًا إلى عميل لاختبار الكود الذي سنكتبه ومعرفة كيف يعمل. أسهل شيء يمكننا تجربته هو عميل Redis، وذلك لأنه سهل الاستخدام وأيضًا لأن الخادم الذي سنكتبه سيكون متوافقًا مع Redis.

وبالتوافق هنا نعني أن أي عميل يعرف كيفية التواصل مع Redis سيكون قادرًا على التواصل مع خادمنا.

يمكنك تثبيت Redis وعميله من الموقع الرسمي لـ Redis [هنا](https://redis.io/docs/getting-started/installation/).

ثم قم بتجربته على الفور:

```bash
$ redis-cli ping

PONG
```

الخطوة الأخيرة التي يجب عليك القيام بها بعد ذلك هي إيقاف خادم Redis لأن الكود والخادم سيستخدمان نفس المنفذ.

- إذا كنت تستخدم نظام macOS:
```
$ brew services stop redis
```

- إذا كنت تستخدم نظام Linux:
```bash
$ sudo systemctl stop redis
```

### كيف يعمل Redis

دعنا نشرح أولاً كيف يعمل Redis من خلال أمثلة. على سبيل المثال، يتم تخزين السلاسل فيها باستخدام مفتاح وقيمة.

المفتاح سيكون سلسلة، والقيمة ستكون كائن مسلسل من أي نوع مثل مصفوفة أو عدد صحيح أو قيمة منطقية.

```bash
SET admin ahmed
```

```bash
> GET admin

"ahmed"
```

الطريقة التي يستقبل بها Redis هذه الأوامر هي عن طريق بروتوكول التسلسل RESP (Redis serialization protocol).

إذا نظرنا إلى كيفية إرسال `SET admin ahmed` كرسالة مسلسلة إلى Redis، ستكون على النحو التالي:

```bash
*3\r\n$3\r\nset\r\n$5\r\nadmin\r\n$5\r\nahmed
```

ومن أجل تبسيط الأمر أكثر:

```bash
*3
$3
set
$5
admin
$5
ahmed
```

إذا نظرنا إلى RESP، سنجد أنه يخبرنا بمعنى كل رمز:

- '*' يشير إلى أن لدينا مصفوفة بحجم 3. بالتالي، سنقرأ 6 أسطر. كل سطرين يمثلان نوع الكائن وحجمه، والسطر الثاني يحتوي على قيمة الكائن.
- '$' يشير إلى أنه سلسلة بعدد 5 أحرف. لذلك، سيكون السطر التالي يحتوي على 5 أحرف فقط.

عندما نقول `GET admin`، سيعيد نفس الكائن بنفس الشكل باستثناء القيمة المختلفة:

```bash
$5\r\nahmed\r\n
```

```bash
$5
ahmed
```

ما سنفعله هو بناء نفس المسلسل ومن خلاله يمكننا التواصل مع خادمنا باستخدام أي عميل Redis بالضبط مثل Redis.

وبهذا نكون جاهزين للبدء في بناء قاعدة البيانات الخاصة بنا.
